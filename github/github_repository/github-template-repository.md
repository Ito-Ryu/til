# GitHub Template Repository

## 概要
テンプレートリポジトリは、既存のリポジトリをテンプレートとして再利用し、同じディレクトリ構造、ブランチ、ファイルを持つ新しいリポジトリを簡単に作成できるようにする機能である。これにより、開発者は一貫したプロジェクトの開始点を簡単に持つことができる。

## 特徴
*   **構造の再利用**: テンプレートリポジトリのディレクトリ構造、ブランチ、ファイルが新しいリポジトリにコピーされる。
*   **独立した履歴**: テンプレートから作成されたリポジトリのブランチは、テンプレートリポジトリとは関連のない独立したコミット履歴を持つため、直接プルリクエストを作成したり、ブランチ間でマージしたりすることはできない。
*   **GitHub Classroomとの連携**: GitHub Classroomの課題のスタートコードとしてテンプレートリポジトリを利用できる。
*   **制限事項**: Git LFS (Large File Storage) を使用して保存されたファイルはテンプレートリポジトリに含めることができない。

## 設定手順
1.  テンプレートとして使用したいリポジトリを作成するか、既存のリポジトリに移動する。
2.  リポジトリのメインページに移動する。
3.  リポジトリ名の下にある「**Settings（設定）**」タブをクリックする。
4.  「General（全般）」設定ページの「Template repository」のセクションで、「**Template repository**」のチェックボックスを選択（オンに）する。

## メリット
*   **効率的なリポジトリ作成**: 新しいプロジェクトを開始する際に、毎回ゼロからボイラープレート（定型コード）やディレクトリを設定する手間を省き、迅速にリポジトリを立ち上げることが可能である。
*   **一貫性の確保**: チームや組織内でプロジェクトの構造や設定（リンター、テスト設定、CI/CDの設定など）の一貫性を保つことができる。
*   **学習・教育用途**: GitHub Classroomと組み合わせることで、学生や学習者に対して統一されたスタートコードを提供し、課題の配布を容易にする。
